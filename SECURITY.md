# Security

## Reporting a vulnerability

Please email **security@agenttransfer.dev** with details and a proof of concept. Do not open a public issue. You'll get an acknowledgment within 72 hours; fixes ship before disclosure.

## Model (what protects what)

| Surface | Protection |
|---|---|
| API keys / admin token | random 256-bit, stored sha256-hashed, rotation endpoint (`rotate-key`) |
| Share links | 128-bit random tokens, server-enforced TTL ≤ `MAX_TTL`, revocation severs in-flight downloads, burn-after-read is single-flight |
| Uploads | per-file cap, per-agent storage quota (reduced tier until the owner verifies) and daily rate, streamed hashing (constant memory), slow-body read deadline (`UPLOAD_BODY_TIMEOUT` — downloads stay untimed) |
| Storage exhaustion | global free-space reserve (`DISK_RESERVE`, default 10%): API uploads and app deploys refuse with 507 while the reserve is active; unverified agents' files expire within `UNVERIFIED_FILE_TTL` (24h) until a human vouches; one owner email can human-verify at most `MAX_AGENTS_PER_OWNER` agents (unproven nominations do not count); app source + expanded release + observed persistent `/data` are checked against `APP_STORAGE_QUOTA`; the janitor stops a running app that later grows over quota; `GET /v1/admin/storage` shows folder/app consumers |
| Outbound email | disabled until the agent's human owner is verified (open-signup instances) — owner CCs included; verification is a **confirm-button POST**, so scanner-prefetched links can't approve; even verified, each agent can only ever email a small circle of unique remote recipients (`HUMAN_RECIPIENTS_MAX`, owner exempt); per-agent daily send rate; relay-only (instance IP never sends); every human-bound mail carries an HMAC-signed unsubscribe link and suppressed addresses are skipped |
| Signup names | open signup can't claim reserved localparts (`postmaster`, `no-reply`, `abuse`, `self`, …); taken names get a random suffix instead of enabling squatting games |
| Inbound email | recipients must exist, size-capped at the socket, DKIM verified and surfaced — offers are `trusted` only on a pass whose signing domain exactly equals the From domain; manifest URLs are stored, **never fetched server-side** (no SSRF) |
| CLI downloads | implicit manifest/`Content-Disposition` names are reduced to one safe visible basename and installed without replacing an existing entry; encrypted downloads use the same path; only explicit `-o` authorizes another path/replacement |
| Resident concierge | acts only on API-authenticated same-instance (`dkim=local`) messages and trusted offers; downloads only from the exact configured instance origin, rejects redirects, re-checks the concrete dial IP, permits loopback only for a loopback-configured origin, caps actual bytes at 64 MiB by default, and has a two-minute whole-request timeout |
| Receipts | ed25519-signed and hash-chained; a supplied genesis-to-head export exposes edits, reordering, and internal gaps; public key published at `/.well-known/agenttransfer`; no external checkpoint means suffix omission and operator re-signing remain possible |
| Signup (open mode) | per-IP rate limit + owner email verification + per-owner agent cap |
| App eligibility | `APP_DOMAIN` must be explicitly enabled and the current owner mailbox must complete a POST-backed email challenge (`owner_verification_method=email`); operator/admin and migrated `legacy` verification have separate provenance and cannot unlock hosting |
| Static app hosts | exact one-label host routing, active-release + owner gate on every request, GET/HEAD only, immutable content-addressed blobs, traversal/link/device/duplicate/decompression limits, sha256 ETags; on-demand TLS issuance is allowed only for an active eligible app |
| App runner control | separate `agenttransfer app-runner` process; mandatory random bearer token of at least 32 bytes over a local Unix socket (setup generates 256 random bits); typed JSON operations and validated IDs/paths/images/argv; only the runner invokes Docker; each proxy request obtains fresh runner attestation that the stored runtime/upstream still match |
| Dynamic app containers | disabled until the operator declares an enforced `/data` filesystem quota (`APP_DATA_QUOTA_ENFORCED`) or explicitly accepts watchdog-only limits (`ALLOW_UNENFORCED_APP_DATA`); arbitrary Dockerfile builds are separately disabled by default; the opt-in path snapshots mutable input into separate runner scratch, serializes and times out builds, restricts remote build sources, and uses operator-selected `none`/`bridge` `RUN` networking; external images/base images require a sha256 digest from an exact registry allowlist; runtime is nonroot, read-only root, all capabilities dropped, `no-new-privileges`, bounded CPU/memory/swap/PIDs and `/tmp`, only app-specific `/data` writable, image-declared volumes outside canonical `/data` and `/tmp` rejected, rotating logs, and 2xx health-gated; egress-off apps use an exclusive labeled internal bridge and an RFC1918 endpoint validated against its exact private IPAM subnet, while egress-enabled apps use a random `127.0.0.1` published port; global plus per-app proxy caps prevent one streaming app from consuming every slot |
| App lifecycle cleanup | stop removes serving but retains release/data; ordinary reset removes releases/runtimes while retaining id/slug and `/data`, explicit purge removes all four; activation and the janitor reconcile stale managed runtimes to SQLite's desired runtime; agent deletion refuses to commit if a container runner is required but unavailable, preventing orphan state; app deploy/stop/delete attempt signed audit receipts |
| Public pages (`/f/`, `/u/`, index) | per-IP rate limit (IPv6 keyed by /64 — full addresses would hand out 2^64 free identities) with an automatic 15-minute ban for repeat hammering; behind a proxy the client IP is read from the proxy-appended XFF hop, never the spoofable first entry |
| Connect instances (anonymous) | outbound email locked until a human owner verifies by magic link; per-instance daily caps on relayed sends and proxied egress; store-and-forward queue caps; suspend kill switch; random non-vanity subdomains; control channel rides the authenticated tunnel and is never publicly routable; client re-verifies DKIM instead of trusting host verdicts. The host still terminates TLS, sees proxied plaintext, and temporarily stores offline raw mail. |
| Metrics | localhost or admin-token gated, never public by default |

## Known gaps (documented, not hidden)

- **No encryption at rest** — run on an encrypted volume if your threat model needs it. The blob directory and SQLite file are plaintext.
- **Sealed-recipient discovery is TOFU, not a transparency log.** The CLI pins each recipient's first observed X25519 key and refuses later changes, and derives the sender key locally. An active instance operator can still substitute the first key before a correct pin exists. Use symmetric encryption with a key exchanged over an independent authenticated channel when first-contact directory trust is unacceptable.
- **Receipt append is not transactionally coupled to every side effect.** Supported successful operations attempt to append a signed receipt and log failures, but a receipt-storage failure cannot undo bytes already served, email already relayed, or an activated runtime. The chain proves only supplied records; it is not an externally anchored, complete action ledger.
- **Docker is isolation, not a tenant-grade VM boundary.** Keep Docker and the host kernel patched. Runtime egress is disabled by default, but a malicious image still attacks the container/kernel boundary; enabling `APP_RUNTIME_EGRESS` also exposes whatever the host network permits. Use separate VMs or a stronger sandbox for mutually hostile tenants.
- **An internal Docker bridge is not a one-way firewall from the host.** It removes the container's external default route, but Docker bridge/host reachability is platform-dependent and host services bound to that bridge may still be reachable. Keep sensitive host daemons off those interfaces or add host firewall policy; use a VM boundary when that distinction matters. Docker Desktop does not route its private bridge to the macOS host, so local Desktop runners need the explicit `APP_RUNTIME_EGRESS=true` loopback-publish mode.
- **Hosted app traffic is not covered by the control-plane IP limiter.** Static hosts are cheap immutable reads; container apps are arbitrary public HTTP services and must implement their own authentication/rate limits. Put a CDN or edge limiter in front when public bandwidth abuse matters.
- **App environment variables are not a secrets manager.** Values are omitted from SQLite, API responses, receipts, and normal CLI/MCP output, but they are visible to the container, Docker metadata, the in-memory deploy path, and a privileged host operator. Use scoped, rotatable credentials.
- **Docker storage is partly outside the blob quota.** Persistent `/data` is measured and enforced, but image layers/build cache are shared Docker state rather than exact per-agent charges. `DISK_RESERVE` gates API writes; it cannot prevent Docker or a running process from consuming host space between checks. Monitor the volume and Docker cache.
- **Persistent `/data` is measured, not hard-quota-mounted.** The janitor stops a runtime after observed use exceeds its budget, but a fast writer can consume bytes or inodes between sweeps. Put `APP_DATA_ROOT` on a separately bounded/quota-enabled filesystem for untrusted tenants. An over-quota volume has no agent shell or file browser; the agent's only self-service recovery is destructive purge, while an operator can preserve/export or reduce `APP_DATA_ROOT/<app-id>`.
- **The shipped backup briefly takes the public service offline.** It stops HTTP/SMTP for the SQLite + immutable-blob hardlink snapshot so references cannot race garbage collection, then restarts before archive compression. Containers continue running and `/data` is only a best-effort live copy—not a point-in-time snapshot. Monitor the logged pause and use application-native quiescing/dumps for consistent app data.
- **No virus scanning** — if you accept uploads from strangers, front `/u/` and `/v1/files` with scanning (e.g. ClamAV via a reverse proxy hook).
- **SPF is not checked** on inbound mail (DKIM is). Recorded as `spf: "none"`; treat unsigned mail accordingly.
- **Anyone with a link can download until it expires** — that's the design (unguessable + short-lived + revocable, with a best-effort signed download record); don't put secrets in world-readable links without `once=1`.
- Local mode (`DOMAIN` unset) binds `:8080` without TLS — it's for localhost development, not the open internet.
